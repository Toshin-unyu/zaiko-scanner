# zaiko-scanner

在庫商品のJANコードをスマホカメラで読み取り、登録済みの商品ページへ直接遷移するツール。

## 画面構成

| 画面 | ファイル | 用途 | 認証 |
|---|---|---|---|
| トップ | `index.html` | スキャン画面・管理画面へのリンク | 不要 |
| スキャン画面 | `scan.html` | JANコードを読み取り、登録済み商品ページへの新規タブリンクを表示 | 不要 |
| 商品登録画面 | `admin.html` | JANコードをスキャン or 手入力し、商品ページURLを登録 | 不要（登録先URLはドメインホワイトリストで制限） |
| 登録商品の管理 | `products.html` | 登録済みJANコード・商品ページURLの一覧表示・編集・削除 | 管理キー必須 |
| 許可ドメインの管理 | `domains.html` | 商品ページとして登録可能なドメインの追加・削除 | 管理キー必須 |

## 使い方

1. **スキャン画面**でJANコードを読み取ると、登録済みなら「商品ページを開く」リンクが表示される（タップで新規タブが開く）。未登録なら「このJANコードを登録する」リンクが表示され、商品登録画面へJANコードを引き継げる
2. **商品登録画面**でJANコードをスキャンまたは手入力し、商品ページURLを入力して登録する。カメラ映像には検出したバーコードの位置に枠線が表示され、内容を確認してから登録できる
3. 商品ページURLとして登録できるのは、**許可ドメインの管理**画面で登録済みのドメイン配下のhttps URLのみ
4. 登録済みデータの修正・削除は**登録商品の管理**画面、許可ドメインの追加・削除は**許可ドメインの管理**画面から行う（いずれも管理キーが必要）

## 構成

- `index.html` / `scan.html` / `admin.html` / `products.html` / `domains.html` / `config.js`: GitHub Pagesで配信するフロントエンド
- `gas/`: バックエンドAPI（GAS Web App）。スプレッドシートに以下2シートを保持
  - `マスタ`: JANコード・商品ページURL・登録日時
  - `許可ドメイン`: 商品ページURLとして登録可能なドメイン一覧

## セキュリティ

- 商品ページURLの登録・更新は、許可ドメイン一覧に含まれる`https`のURLのみ受け付ける（オープンリダイレクト対策）
- ドメイン追加・削除、商品情報の更新・削除には管理キーが必要（`PropertiesService`にサーバー側で保存し、タイミングセーフ比較で検証。コードやリポジトリに値は含まれない）
- 書き込みAPI（`doPost`）全体を`LockService`で排他制御し、同時操作による競合を防止

## デプロイ

- **フロントエンド**: GitHub Pages（`main`ブランチのルートを配信）。リポジトリはPublic（Organizationの契約プラン上、Private repoではPagesが使えないため）
- **バックエンド**: `cd gas && clasp push --force` → `clasp deploy --deploymentId <既存デプロイID>` で更新。デプロイURLは固定（`config.js`の`GAS_API_URL`）なので、通常はデプロイID指定の更新で足りる
- 管理キーは`PropertiesService`にのみ保存されており、リポジトリには含まれない（初回設定時のみ別途対応が必要）
